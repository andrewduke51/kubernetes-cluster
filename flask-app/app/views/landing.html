Run Kubernetes Pod with root privileges (not recommended)

Most Docker containers and the processes inside run with a non-root user because of better security. If the container process is running with root (uid 0), it will be the same root as on the host. In this case, the user may get access to the host from the container, thus gaining root privilege on the host.
